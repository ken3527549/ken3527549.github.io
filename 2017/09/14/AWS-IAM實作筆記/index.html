IAM筆記

IAM功能

  • 管理多個IAM使用者及其存取權。
  • 管理聯合身分使用者的存取權。
  • 所有 AWS 客戶都可使用 IAM。此服務並不收取額外費用。
  • IAM使用者可以向 Amazon S3 和 EC2 等 Web 服務提出請求。

政策(policy)

IAM使用者存取資源的權限設定被稱為政策(policy)。

  • IAM policy Simulator

IAM角色(role)

  • 角色和使用者或群組無關，他是一個獨立的實體身分。
  • 透過將定義好的存取權(permission)委派給角色，IAM使用者、不同 AWS 帳戶下的 IAM 使用者或EC2之類的AWS服務，可以利用角色權限執行功能。

實作流程

  1. Create groups of users.
  2. Set permissions for each group.
  3. Generate a password.
  4. Sign into the AWS Management Console as an IAM.

MFA(AWS Multi-Factor-Authertication)
